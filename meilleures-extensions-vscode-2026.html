Les meilleures extensions VS Code en 2026 : ma sélection

VS Code reste l'IDE le plus utilisé au monde. Sa puissance vient en grande partie de son écosystème d'extensions. Voici celles que j'utilise vraiment au quotidien — pas de bloat, que des outils qui font gagner du temps.

Extensions IA — Indispensables en 2026

1. Codeium (gratuit)

La meilleure alternative gratuite à GitHub Copilot. Complétion multi-lignes, chat, et maintenant "Cascade" (agent autonome). Aucune limite artificielle sur le plan gratuit pour les individus.

Installer : ext install Codeium.codeium

2. Continue (gratuit, open-source)

Un copilote IA open-source qui se connecte à n'importe quel modèle : Claude, GPT-4, Ollama local, Mistral. Parfait pour ceux qui veulent garder le contrôle sur quel modèle ils utilisent.

Installer : ext install Continue.continue

Git — Ne jamais toucher à la ligne de commande

3. GitLens

Le meilleur outil Git pour VS Code. Blame inline, historique visuel, comparaison de branches. La version gratuite est très complète.

Installer : ext install eamodio.gitlens

4. Git Graph

Visualisation du graph de commits en arbre. Simple, performant, essentiel pour comprendre l'historique d'un projet complexe.

Installer : ext install mhutchie.git-graph

Qualité de code

5. ESLint / Prettier

Incontournables pour JavaScript/TypeScript. Configurez-les une fois, oubliez le formatage pour toujours.

ext install dbaeumer.vscode-eslint
ext install esbenp.prettier-vscode

6. Error Lens

Affiche les erreurs et warnings directement sur la ligne de code, sans avoir à aller dans le panneau "Problèmes". Change complètement la façon de débugger.

Installer : ext install usernamehw.errorlens

Productivité générale

7. TODO Highlight

Colore les TODO, FIXME, HACK dans votre code. Pratique pour retrouver rapidement les notes laissées en cours de dev.

8. Path Intellisense

Autocomplétion des chemins de fichiers dans vos imports. Indispensable dès qu'on travaille sur un projet avec une structure de fichiers complexe.

9. Remote - SSH

Éditez des fichiers sur un serveur distant directement dans VS Code, comme si c'était en local. Terminal intégré, intellisense, tout fonctionne à distance.

Installer : ext install ms-vscode-remote.remote-ssh

10. Docker

Interface graphique pour gérer vos containers Docker directement dans VS Code. Voir les logs, arrêter/démarrer des services — sans jamais quitter l'éditeur.

Thèmes — Parce que l'esthétique compte

One Dark Pro

Le thème le plus populaire de VS Code. Couleurs équilibrées, lisible pendant des heures sans fatiguer les yeux.

Catppuccin

Pour ceux qui veulent quelque chose de plus pastel et moderne. 4 variantes (Latte, Frappé, Macchiato, Mocha).

Mon setup exact

Si vous voulez copier mon setup complet, voici le fichier extensions.json à mettre dans .vscode/ :

{
  "recommendations": [
    "Codeium.codeium",
    "Continue.continue",
    "eamodio.gitlens",
    "mhutchie.git-graph",
    "dbaeumer.vscode-eslint",
    "esbenp.prettier-vscode",
    "usernamehw.errorlens",
    "christian-kohler.path-intellisense",
    "ms-vscode-remote.remote-ssh",
    "ms-azuretools.vscode-docker"
  ]
}

Partagez ce fichier avec votre équipe et tout le monde aura les mêmes extensions installées.
